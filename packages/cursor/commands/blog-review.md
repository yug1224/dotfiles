ブログ記事評価プロンプトに基づいて、指定された記事を7つの観点から評価し、建設的で実践的なフィードバックを提供する。

**参照ルール**: `~/.cursor/rules/blog-review-rule.mdc`

**Input**: `/blog-review` の後に続く引数は、評価対象の記事ファイルパスまたは URL。

---

## Steps

1. **評価基準の読み込み**

   `~/.cursor/rules/blog-review-rule.mdc` を Read ツールで読み込み、評価基準・スコア定義・レポート形式を把握する。

2. **記事の取得**

   引数で指定された記事を取得する。ファイルパスの場合は Read ツール、URL の場合は Web Fetch で取得する。
   引数が未指定の場合はユーザーに入力を求める。

3. **評価の実施**

   読み込んだ評価基準に基づき、以下の7つの観点で記事を評価する。

   - 防御力 (0.0-5.0)
   - 思考整理力 (0.0-5.0)
   - 実践応用性 (0.0-5.0)
   - 構成と読みやすさ (0.0-5.0)
   - コミュニケーション力 (0.0-5.0)
   - 炎上リスク・社会的配慮 (0.0-5.0)
   - 人間らしさ (0.0-5.0)

4. **レポート出力**

   評価基準内の「評価レポート形式」に従って Markdown でレポートを出力する。
   優先改善提案 TOP3 とリライト例を含める。

---

## Guardrails

- 評価基準ファイルが読み込めない場合はユーザーに通知する
- 記事が未指定の場合はユーザーに入力を求める
- スコアは厳格に付ける（5.0 は例外的に優れた記事のみ）
- 迷った場合は低い方のスコアを付ける
- 企業ブログ・組織名が紐づく場合は1段階厳しく評価する
