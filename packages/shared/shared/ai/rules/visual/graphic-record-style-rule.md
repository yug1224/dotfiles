# グラフィックレコード画像 — スタイル仕様（固定）

**v3.1**: 日本語自然文・1回貼り付け。ブロックAは `image_generation_prompt`（日本語描画内容）+ 【画風・レイアウト】の必須装飾指示。`compressed_summary` の markdown fenced は廃止。下流は **Gemini Nano Banana Pro**（Thinking + 画像を作成）を正とする。v3.1 は鮮やかフラット・装飾強化・文章厚化（項目数12は維持）。

画像生成 LLM 向けの画風・レイアウト・禁止事項の正本。`/graphic-record-prompt` は **ブロックA（画像生成用）** に内容を流し込む。

## 下流 LLM 向け契約

`/graphic-record-prompt` の出力は **2 段階**（プロンプト生成 → 別チャットで画像生成）。

- 下流の正: **Gemini アプリ**で「画像を作成」+ **Thinking モデル（Nano Banana Pro）**
- Step2 では **fenced ブロック 1 本を 1 回で**貼る（2通に分割しない）
- 矛盾時の優先: **固定セクション（出力指示・画風・避ける要素）** ＞ **`image_generation_prompt` の描画内容**

## ブロックA（画像生成用・固定文案）

`/graphic-record-prompt` は次の **固定文** をそのまま出力し、`{image_generation_prompt}` だけを差し替える（言い換え・行の追加・削除禁止）:

```text
【出力指示】
16:9のインフォグラフィック画像を1枚だけ生成してください。説明文・箇条書きのテキスト応答は不要です。画像のみ出力してください。

【画風・レイアウト】
白背景、16:9横長（1920x1080）、鮮やかで読みやすいデジタルフラットイラスト風の日本語グラフィックレコード（インフォグラフィック調。写真リアル・手書き風・ベタ塗り単色のみは避ける）。
画面上部中央に太字タイトル帯（グラデーション帯・太枠）。左から4列: 1.背景（鮮やかな青のグラデーション色面・狭い）/ 2.目標（鮮やかな緑のグラデーション色面・狭い）/ 3.実装（黄オレンジのグラデーション色面・列幅最大）/ 4.残課題（赤またはグレーのグラデーション色面・列幅最小）。
各列見出しにグラデーション帯・太枠・小アイコン・番号バッジ。列間に太めの色付き矢印またはブリッジ（光沢ハイライト可）で左→右の流れを示す。角丸ボックス（軽いドロップシャドウ・細いハイライト縁）・番号バッジ・リボン・星/メダル装飾・チェック/×印・キャプション枠付きミニ図解を各列で多用する。
背景列: ミニ図解3組、赤×または鮮やかな青矢印で課題を示す。目標列: ミニ図解3組、緑チェック・金/緑バッジで達成状態。実装列: 小見出し帯2つ以上、黄コールアウト枠2つ以上、表1つ+フロー1つ以上、アイコン多め。残課題列: 工事バリア/×印/点線矢印で「今回やらない」を視覚化。
ボックスと余白で区切り、各図解に読める日本語キャプション（名詞+説明文）を付ける。文字量は多めだが1920x1080で判読可能なサイズを維持。左下にメガネをかけた熊 + 吹き出し1つ。熊以外のマスコットは禁止。

【描画内容】
{image_generation_prompt}

【避ける要素】
マーカー, 消しゴム, ペン, ペン立て, 手, 会議室, 机, 額縁, 枠線, 写真リアル, 暗い背景, 透かし, ロゴ, 判読不能な過密文字, 誤った日本語, 文字化け, 英数字だけのラベル, 単語だけの散在テキスト, 画面下部の番号付きフロー, ステップ列, 上から4段の横長帯, 縦積みレイアウト, 縦書き列名, 英語だけのキャプション
```

- `{image_generation_prompt}` には `graphic-record-source-rule` で生成した日本語描画内容を入れる
- コマンド引数の `size:WIDTHxHEIGHT` は 【画風・レイアウト】の解像度行（`1920x1080`）を上書きする
- **`【実行指示 — 厳守】` は使わない**（下流で拒否されやすい）

## ネガティブプロンプト（固定）

次の **1 行を正本**として保持する。ブロックAの 【避ける要素】 に統合済み（別出力は不要）:

```text
マーカー, 消しゴム, ペン, ペン立て, 手, 会議室, 机, 額縁, 枠線, 写真リアル, 暗い背景, 透かし, ロゴ, 判読不能な過密文字, 誤った日本語, 文字化け, 英数字だけのラベル, 単語だけの散在テキスト, 画面下部の番号付きフロー, ステップ列, 上から4段の横長帯, 縦積みレイアウト, 縦書き列名, 英語だけのキャプション
```

## 画風（固定）

**鮮やかな日本語グラフィックレコード**の 1 枚イラスト。**デジタルフラットイラスト風で統一**（手書き風・写真リアル・「または」併記は使わない）。インフォグラフィック調で視覚インパクトを出す。

### 含める

- **背景**: **白**
- **列の色面**: 背景=鮮やかな青のグラデーション、目標=鮮やかな緑のグラデーション、実装=黄オレンジのグラデーション、残課題=赤またはグレーのグラデーション（白背景とのコントラスト強め）
- **線・文字**: 太めで読みやすい。文字量は多めだが判読可能なサイズを維持
- **装飾**: フラットアイコン、色付きバッジ、リボン、星/メダル、チェック、表・フロー（主に実装列）、軽いドロップシャドウ・ハイライト縁
- **図解**: 角丸ボックス、太めの色付き矢印（実線/点線）、キャプション枠付きミニ図解
- **左下**: かわいい熊（丸眼鏡）+ 吹き出し 1 つ（ユーモアあるセリフ）

### 図解密度（列ごと）

- **背景・目標**: 各列にミニ図解 **3 組**（フラットアイコン + キャプション + 青矢印／赤×）
- **実装**: **列幅最大**。**表 1 つ + フロー 1 つ以上を必須**。小見出し帯・黄コールアウト枠・番号バッジ・複数ボックスを積極利用（最大 6 項目相当）
- **残課題**: **列幅最小**。1〜2 項目をコンパクトに。×印・点線矢印でスコープ外を明示

### 装飾（必須）

- **全列共通**: 列見出しのグラデーション帯 + 太枠 + 小アイコン + 番号バッジ、列間の太め矢印/ブリッジ、角丸ボックス（軽い影）、キャプション枠、各列に星/メダル/リボン装飾を 1 つ以上
- **背景**: 赤×または鮮やかな青矢印で Before/課題
- **目標**: 緑チェック・金/緑バッジで達成状態
- **実装**: 小見出し帯 2 つ以上、黄コールアウト枠 2 つ以上、表 + フロー必須
- **残課題**: 工事バリア/×印/点線矢印で「今回やらない」を視覚化
- 名詞+説明文のキャプションは必須。**単語だけの羅列は禁止**（ネガティブの「単語だけの散在テキスト」に該当）

### 含めない

- マーカー、消しゴム、ペン立て、磁石、付箋の実物、手、人物の全身
- 額縁・机・椅子、会議室・写真風の小物や景深
- **上から積む横長帯4段**（縦積みレイアウト）
- **画面下部の番号付きフロー・左→右のステップ列**

## カラーパレット（参考）

次の箇条書きを **この順で常に**使う:

- 用紙: 白
- 背景列: ライトブルー〜ビビッドブルーのグラデーション色面 + 鮮やかな青の矢印・太枠
- 目標列: ライトグリーン〜ビビッドグリーンのグラデーション色面 + 鮮やかな緑のバッジ・チェック・金メダル
- 実装列: ライトイエロー〜オレンジのグラデーション色面 + 強調枠（**列幅最大**）
- 残課題列: ライトレッドまたはライトグレーのグラデーション色面 + 赤い×印（**列幅最小**）
- 本文ラベル: 黒または濃いグレー（太字）

## キャンバス（参考）

- 16:9 横長（デフォルト **1920×1080 px**。`size:WIDTHxHEIGHT` で上書き時はその値）
- 背景: **白**
- 画風: 鮮やか・グラフィックレコード、**デジタルフラットイラスト風で統一**（小物なし・インフォグラフィック調）

## 構図（固定骨格）

テンプレ ID や英字スラッグは書かない。次の骨格を正とする:

- **画面上部中央**: 見出しタイトル（太字・単独帯）
- **その下**: **左から 4 列** — `【1.背景】` / `【2.目標】` / `【3.実装】` / `【4.残課題】`
- **3.実装** が **最も列幅が広い**（表・フロー・複数ボックス可）
- **4.残課題** が **最も列幅が狭い**
- **左下**: マスコット専用（4 列エリアと重ねない）
- **禁止レイアウト**: 上から積む横長帯 4 段、背景と目標だけの上下2段

### 列幅の目安（参考）

| 列         | 目安の幅割合         |
| ---------- | -------------------- |
| タイトル帯 | 画面上部 8〜10% 高さ |
| 背景       | 約 15〜18%           |
| 目標       | 約 15〜18%           |
| 実装       | 約 45〜50%           |
| 残課題     | 約 12〜15%           |

## 画面構成（`compressed_summary` 内部 QA 用）

`image_generation_prompt` では **列見出し**（`【1.背景】` / `【2.目標】` / `【3.実装】` / `【4.残課題】`）を使う。本節は `compressed_summary` 組み立て時の列上限参考。見出し形式は source ルール「compressed_summary の組み立て」を正とする。

**背景 + 目標 + 実装 + 残課題の合計は 12 以内**（超える場合は **背景 → 目標 → 実装** の順で削る。実装は最優先で残す）。

| 列     | 上限       | 列幅     |
| ------ | ---------- | -------- |
| 背景   | **3 項目** | 狭       |
| 目標   | **3 項目** | 狭       |
| 実装   | **6 項目** | **最大** |
| 残課題 | **2 項目** | **最小** |

## 密度・表記

- 載せきれない詳細は削る。優先は **key_messages 3〜5 個**（主に実装へ配分）
- `image_generation_prompt`: 各列に **列リード 1 文**（50〜90 字）、各項目は **ミニ図解説明 + キャプション「名詞+説明文」**（70〜110 字/項目目安）
- 全体文字数 **2000〜2800 字**。読める密度の文字は推奨（判読不能な過密のみ禁止）
- サマリ本文: 日本語主体。短い技術識別子（`` `list()` ``、`` `staleTime` `` 等）は **実装列の図・表内** に限り可
- パス全文・diff 行・長い英語段落は禁止

### キャプション・見出し（描画）

- **列リード**: 各列 1 文（50〜90 字）で列の主旨と文脈を説明
- **画像内固定文言**: **「」で囲む**（タイトル・キャプション・コールアウト・熊のセリフ）
- **実装列の図・表内**: 短い技術識別子可（参考 ADR 画像準拠）
- 英語だけのキャプション・名詞だけの羅列は禁止

## マスコット（固定）

- 左下: 熊（丸眼鏡）+ 吹き出し 1 つ
- セリフ: **20 字前後**、ユーモア可（`mascot_line` を `熊の吹き出し「」` に入れる）

## 禁止（スタイル全体）

- 写真リアル、暗い背景、透かし・ロゴ、マスコット複数（右上+左下など）
- クリーム〜ベージュのみの用紙指定（正は **白**）
- 上から積む横長帯 4 段
- 画面下部の番号付きフロー・ステップ列
- `【実行指示 — 厳守】` 形式の出力指示（v2.x 互換。拒否されやすい）
- 上記「含めない」要素以外の実景メタファ
