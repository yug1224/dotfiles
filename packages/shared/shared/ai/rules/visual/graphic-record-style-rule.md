# グラフィックレコード画像 — スタイル仕様（固定）

画像生成 LLM 向けの画風・レイアウト・禁止事項の正本。`/graphic-record-prompt` は **ブロックA（画像生成用）** に内容を流し込む。

## 下流 LLM 向け契約

`/graphic-record-prompt` の出力は **2 段階**（プロンプト生成 → 別チャットで画像生成）。下流は Cursor / ChatGPT / Gemini 等を想定する。

- **固定実行指示 + `markdown` サマリ** の fenced ブロック 1 本だけを Step2 に貼る。これが画像生成の実行仕様
- 矛盾時の優先: **固定指示** ＞ **サマリ markdown**

## ブロックA（画像生成用・固定文案）

`/graphic-record-prompt` は次の **固定文** をそのまま出力し、`compressed_summary` だけを差し替える（言い換え・行の追加・削除禁止）:

````text
日本語にてグラフィックレコードで説明している画像に変換してください。
図や矢印、ボックス、キャプション、色を使って、コアとなるアイデアを視覚的に説明してください。

背景色は白。正しい日本語を出力してください。
可愛い動物さん（メガネを掛けた熊さん）を左下に、ワンポイントだけ配置してください。
熊はユーモアあふれるセリフを表示してください。

左から順番に 1.背景・2.目標・3.実装・4.残課題 とグルーピングしてまとめてください。
3.実装 は他と比べて広くしてください。

```markdown
{compressed_summary}
````

````

- 内側の fenced `markdown` には `graphic-record-source-rule` で生成した `compressed_summary` を入れる
- 追いメッセージは **任意**（参考運用ではブロックAのみで成功）

## ネガティブプロンプト（固定）

次の **1 行を固定**（レガシー向け。ブロックA出力には含めない）:

```text
マーカー, 消しゴム, ペン, ペン立て, 手, 会議室, 机, 額縁, 枠線, 写真リアル, 暗い背景, 透かし, ロゴ, 文字詰め込み, 誤った日本語, 文字化け, 英数字だけのラベル, 単語だけの散在テキスト, 画面下部の番号付きフロー, ステップ列, 上から4段の横長帯, 縦積みレイアウト, 縦書き列名, 英語だけのキャプション
````

## 画風（固定）

**カラフルな日本語グラフィックレコード**の 1 枚イラスト。**デジタルフラットイラスト風で統一**（手書き風・「または」併記は使わない）。

### 含める

- **背景**: **白**
- **列の色面**: 背景=薄青、目標=薄緑、実装=薄黄〜薄オレンジ、残課題=薄赤または薄グレー
- **線・文字**: 太めで読みやすい
- **装飾**: フラットアイコン、色付きバッジ、チェック、表・フロー（主に実装列）
- **図解**: 角丸ボックス、矢印（実線/点線）、キャプション枠付きミニ図解
- **左下**: かわいい熊（丸眼鏡）+ 吹き出し 1 つ（ユーモアあるセリフ）

### 図解密度（列ごと）

- **背景・目標**: 各列にミニ図解 1〜3 組（フラットアイコン + キャプション + 青矢印／赤×）
- **実装**: **列幅最大**。表・フロー・番号付き小見出し・複数ボックスを積極利用（最大 6 項目相当）
- **残課題**: **列幅最小**。1〜2 項目をコンパクトに

### 含めない

- マーカー、消しゴム、ペン立て、磁石、付箋の実物、手、人物の全身
- 額縁・机・椅子、会議室・写真風の小物や景深
- **上から積む横長帯4段**（縦積みレイアウト）
- **画面下部の番号付きフロー・左→右のステップ列**

## カラーパレット（参考）

次の箇条書きを **この順で常に**使う:

- 用紙: 白
- 背景列: 薄い青の背景 + 鮮やかな青の矢印・枠
- 目標列: 薄い緑の背景 + 鮮やかな緑のバッジ・チェック
- 実装列: 薄い黄または薄オレンジの背景 + 強調枠（**列幅最大**）
- 残課題列: 薄い赤または薄グレーの背景 + 赤い×印（**列幅最小**）
- 本文ラベル: 黒または濃いグレー（太字）

## キャンバス（参考）

- 16:9 横長（デフォルト **1920×1080 px**。`size:WIDTHxHEIGHT` で上書き時はその値）
- 背景: **白**
- 画風: カラフル・グラフィックレコード、**デジタルフラットイラスト風で統一**（小物なし）

## 構図（固定骨格）

テンプレ ID や英字スラッグは書かない。次の骨格を正とする:

- **画面上部中央**: 見出しタイトル（太字・単独帯）
- **その下**: **左から 4 列** — `1.背景` / `2.目標` / `3.実装` / `4.残課題`
- **3.実装** が **最も列幅が広い**（表・フロー・複数ボックス可）
- **4.残課題** が **最も列幅が狭い**
- **左下**: マスコット専用（4 列エリアと重ねない）
- **禁止レイアウト**: 上から積む横長帯 4 段、背景と目標だけの上下2段

### 列幅の目安（参考）

| 列         | 目安の幅割合         |
| ---------- | -------------------- |
| タイトル帯 | 画面上部 8〜10% 高さ |
| 背景       | 約 15〜18%           |
| 目標       | 約 15〜18%           |
| 実装       | 約 45〜50%           |
| 残課題     | 約 12〜15%           |

## 画面構成（抽出・サマリ用）

`###` は **常に次の 4 つ**（順序固定）。該当なしは `- （なし）` 1 行のみ。

| 見出し       | 役割                     | 上限       | 列幅     |
| ------------ | ------------------------ | ---------- | -------- |
| `### 背景`   | 経緯・現状・課題の前提   | **3 項目** | 狭       |
| `### 目標`   | 達成したいこと・成功状態 | **3 項目** | 狭       |
| `### 実装`   | 方針・設計・実装・検証   | **6 項目** | **最大** |
| `### 残課題` | 未完了・フォローアップ   | **2 項目** | **最小** |

**背景 + 目標 + 実装 + 残課題の合計は 12 以内**（超える場合は **背景 → 目標 → 実装** の順で削る。実装は最優先で残す）。各項目は **1 行・40 字目安**。

## 密度・表記

- 載せきれない詳細は削る。優先は **key_messages 3〜5 個**（主に実装へ配分）
- サマリ本文: 日本語主体。短い技術識別子（`` `list()` ``、`` `staleTime` `` 等）は **実装列の図・表内** に限り可
- パス全文・diff 行・長い英語段落は禁止

### キャプション・見出し（描画）

- **列見出し・キャプション・熊のセリフ**: 自然な日本語の短文・句
- **実装列の図・表内**: 短い技術識別子可（参考 ADR 画像準拠）
- 英語だけのキャプション・名詞だけの羅列は禁止

## マスコット（固定）

- 左下: 熊（丸眼鏡）+ 吹き出し 1 つ
- セリフ: **20 字前後**、ユーモア可（`mascot_line` をそのまま）

## 画像API用プロンプト（レガシー・任意）

ブロックA方式では **使わない**。他ツール向けに 1 段落が必要な場合のみ、次を満たす:

- **500〜700 字**、白背景、**左から 4 列**（実装列最広）、縦積み 4 帯は書かない
- キャプションは最大 6 句、列見出しは「背景」「目標」「実装」「残課題」

## 禁止（スタイル全体）

- 写真リアル、暗い背景、透かし・ロゴ、マスコット複数（右上+左下など）
- クリーム〜ベージュのみの用紙指定（正は **白**）
- 上から積む横長帯 4 段
- 画面下部の番号付きフロー・ステップ列
- 上記「含めない」要素以外の実景メタファ
