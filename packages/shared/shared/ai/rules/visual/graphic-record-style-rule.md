# グラフィックレコード画像 — 構造仕様（固定）

構造のみ・トーンは外部。ブロックAは `image_generation_prompt`（日本語描画内容）+ 【レイアウト】の構造指示のみ。画風・装飾トーンは **含めない**（`.local.md` またはユーザーが別メッセージで先に送る）。`compressed_summary` の markdown fenced は廃止。下流は **Gemini Nano Banana Pro**（Thinking + 画像を作成）を正とする。

画像生成 LLM 向けのレイアウト・構図・禁止事項の正本。`/graphic-record-prompt` は **ブロックA（画像生成用）** に内容を流し込む。

## 下流 LLM 向け契約

`/graphic-record-prompt` の出力は **2 段階**（プロンプト生成 → 別チャットで画像生成）。

- 下流の正: **Gemini アプリ**で「画像を作成」+ **Thinking モデル（Nano Banana Pro）**
- Step2 では **2 通に分ける**:
  1. **先に** トーン指示を 1 通送る（`graphic-record-style-rule.local.md` の内容、またはユーザーが手書きした画風・装飾・色味の指示）
  2. **続けて** 本ルールの fenced ブロック（構造のみ）を 1 回で貼る
- トーン（画風・装飾・色の強調）は **本ルールの責務外**。`.local.md` が存在しない場合、ユーザーが Step2 の前にトーンを自分で送る責任を持つ
- 矛盾時の優先: **【出力指示】** ＞ **【レイアウト】** ＞ **【避ける要素】** ＞ **`image_generation_prompt` の描画内容**

## ブロックA（画像生成用・固定文案）

`/graphic-record-prompt` は次の **固定文** をそのまま出力し、`{image_generation_prompt}` だけを差し替える（言い換え・行の追加・削除禁止）:

```text
【出力指示】
16:9の画像を1枚だけ生成してください。説明文・箇条書きのテキスト応答は不要です。画像のみ出力してください。

【レイアウト】
16:9横長（1920x1080）。
画面上部中央にタイトル帯。本文はまとまり: 1.背景 / 2.目標 / 3.実装（実装が最大）。4.残課題は任意（あれば1〜2項目）。
各まとまりに見出し帯・ミニ図解・キャプション枠を配置する。背景・目標は各3組、実装は小見出し帯2つ以上・表1つ+フロー1つ以上・コールアウト枠2つ以上（最大6項目相当）。
各図解に読める日本語キャプション（名詞+説明文）を付ける。判読可能な文字サイズを維持。左下に熊1体 + 吹き出し1つ。熊以外のマスコットは禁止。

【描画内容】
{image_generation_prompt}

【避ける要素】
判読不能な過密文字, 誤った日本語, 文字化け, 英数字だけのラベル, 単語だけの散在テキスト, 画面下部の番号付きフロー, ステップ列, 上から4段の横長帯, 縦積みレイアウト, 縦書き列名, 英語だけのキャプション, マスコット複数, 4まとまりエリアと重なる左下以外のマスコット配置
```

- `{image_generation_prompt}` には `graphic-record-source-rule` で生成した日本語描画内容を入れる
- コマンド引数の `size:WIDTHxHEIGHT` は 【レイアウト】の解像度行（`1920x1080`）を上書きする
- **`【実行指示 — 厳守】` は使わない**（下流で拒否されやすい）

## ネガティブプロンプト（固定）

次の **1 行を正本**として保持する。ブロックAの 【避ける要素】 に統合済み（別出力は不要）:

```text
判読不能な過密文字, 誤った日本語, 文字化け, 英数字だけのラベル, 単語だけの散在テキスト, 画面下部の番号付きフロー, ステップ列, 上から4段の横長帯, 縦積みレイアウト, 縦書き列名, 英語だけのキャプション, マスコット複数, 4まとまりエリアと重なる左下以外のマスコット配置
```

## 構図（固定骨格）

テンプレ ID や英字スラッグは書かない。次の骨格を正とする:

- **画面上部中央**: 見出しタイトル（単独帯）
- **その下**: **まとまり** — `【1.背景】` / `【2.目標】` / `【3.実装】`（必須）、`【4.残課題】`（**任意**。内容がなければ省略）
- **3.実装** が **最も面積が広い**（表・フロー・複数ボックス可）
- **左下**: マスコット専用（4 まとまりエリアと重ねない）
- **禁止レイアウト**: 上から積む横長帯 4 段、背景と目標だけの上下2段

### まとまりの面積目安（参考）

| まとまり   | 目安の割合           |
| ---------- | -------------------- |
| タイトル帯 | 画面上部 8〜10% 高さ |
| 背景       | 約 15〜18%           |
| 目標       | 約 15〜18%           |
| 実装       | 約 45〜55%           |
| 残課題     | 約 12〜15%（任意）   |

## 図解密度（まとまりごと）

- **背景・目標**: 各 **3 組**（ミニ図解 + キャプション）
- **実装**: **面積最大**。**表 1 つ + フロー 1 つ以上を必須**。小見出し帯・コールアウト枠・複数ボックス（最大 6 項目相当）
- **残課題**（任意）: あれば 1〜2 項目をコンパクトに

## 画面構成（`compressed_summary` 内部 QA 用）

`image_generation_prompt` では **まとまり見出し**（`【1.背景】` / `【2.目標】` / `【3.実装】`、および任意の `【4.残課題】`）を使う。本節は `compressed_summary` 組み立て時の上限参考。見出し形式は source ルール「compressed_summary の組み立て」を正とする。

**背景 + 目標 + 実装 + 残課題（あれば）の合計は 12 以内**（超える場合は **背景 → 目標 → 実装** の順で削る。実装は最優先で残す）。

| まとまり | 上限       | 面積     |
| -------- | ---------- | -------- |
| 背景     | **3 項目** | 狭       |
| 目標     | **3 項目** | 狭       |
| 実装     | **6 項目** | **最大** |
| 残課題   | **2 項目** | 任意     |

## 密度・表記

- 載せきれない詳細は削る。優先は **key_messages 3〜5 個**（主に実装へ配分）
- `image_generation_prompt`: 各列に **列リード 1 文**（50〜90 字）、各項目は **ミニ図解説明 + キャプション「名詞+説明文」**（70〜110 字/項目目安）
- 全体文字数 **2000〜2800 字**。読める密度の文字は推奨（判読不能な過密のみ禁止）
- サマリ本文: 日本語主体。短い技術識別子（`` `list()` ``、`` `staleTime` `` 等）は **実装列の図・表内** に限り可
- パス全文・diff 行・長い英語段落は禁止

### キャプション・見出し（描画）

- **列リード**: 各列 1 文（50〜90 字）で列の主旨と文脈を説明
- **画像内固定文言**: **「」で囲む**（タイトル・キャプション・コールアウト・熊のセリフ）
- **実装列の図・表内**: 短い技術識別子可（参考 ADR 画像準拠）
- 英語だけのキャプション・名詞だけの羅列は禁止

## マスコット（配置のみ）

- **位置**: 左下（4 まとまりエリアと重ねない）
- **構成**: 熊 1 体 + 吹き出し 1 つ（熊以外のマスコットは禁止）
- **セリフ**: `mascot_line` を `熊の吹き出し「」` として `image_generation_prompt`（【描画内容】）に入れる
- **見た目・画風**: 本ルールでは指定しない（トーン指示または `.local.md` の責務）
