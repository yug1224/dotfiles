# OWASP Top 10 チェックリスト

セキュリティレビュー・設計時に参照する OWASP Top 10 (2021) の各項目に対応するチェックリストと対策パターン。

## A01: Broken Access Control（アクセス制御の不備）

- [ ] 認可チェックが全エンドポイントで実施されているか
- [ ] ロールベースまたはポリシーベースのアクセス制御が適用されているか
- [ ] マルチテナント環境でテナント識別子によるデータ分離が行われているか
- [ ] CORS の設定が最小限に制限されているか
- [ ] ディレクトリトラバーサルが防止されているか

**対策パターン:** Guard / Middleware でエンドポイントごとに認可チェック。テナント識別子を Infrastructure 層で付与。CORS はホワイトリスト方式。

## A02: Cryptographic Failures（暗号化の失敗）

- [ ] 機密データが転送中・保存時に暗号化されているか
- [ ] パスワードが適切なハッシュアルゴリズム（bcrypt, argon2）で保存されているか
- [ ] 暗号化キーが安全に管理されているか
- [ ] 廃止された暗号アルゴリズム（MD5, SHA-1）を使用していないか

**対策パターン:** パスワードは bcrypt（コストファクター 12+）/ argon2。データ暗号化は AES-256-GCM。キー管理は環境変数 / Vault / KMS。

## A03: Injection（インジェクション）

- [ ] SQL クエリがパラメータ化されているか（ORM 使用推奨）
- [ ] ユーザー入力がサニタイズされているか
- [ ] NoSQL インジェクションが防止されているか
- [ ] OS コマンドインジェクションが防止されているか

**対策パターン:** ORM のパラメータ化クエリ。入力バリデーションライブラリ（class-validator, zod 等）。ユーザー入力を OS コマンドに直接渡さない。

## A04: Insecure Design（安全でない設計）

- [ ] 脅威モデリングが実施されているか
- [ ] ビジネスロジックの認可チェックが適切か
- [ ] レート制限が設定されているか
- [ ] エラーメッセージが過剰な情報を漏洩していないか

**対策パターン:** 新機能追加時に脅威モデル作成。レート制限を API レベルで適用。エラー詳細はログのみ、クライアントには汎用メッセージ。

## A05: Security Misconfiguration（セキュリティの設定ミス）

- [ ] デフォルト認証情報が変更されているか
- [ ] 不要なサービス・ポートが無効化されているか
- [ ] セキュリティヘッダーが適切に設定されているか
- [ ] エラーページがスタックトレースを表示していないか
- [ ] 本番環境でデバッグモードが無効化されているか

**対策パターン:** Helmet でセキュリティヘッダー一括設定。環境別設定ファイルで本番/開発を分離。

## A06: Vulnerable and Outdated Components（脆弱で古いコンポーネント）

- [ ] 依存パッケージに既知の脆弱性がないか（`npm audit`）
- [ ] 定期的な依存関係の更新プロセスがあるか
- [ ] 使用していない依存パッケージが削除されているか

**対策パターン:** `npm audit` / `pnpm audit` を CI に組み込む。Dependabot / Renovate で自動更新。

## A07: Identification and Authentication Failures（識別と認証の失敗）

- [ ] パスワードポリシーが適切に設定されているか
- [ ] ブルートフォース攻撃への対策があるか（アカウントロック、レート制限）
- [ ] セッション管理が適切か（タイムアウト、再生成）
- [ ] MFA が利用可能か

**対策パターン:** ログイン試行にアカウントロック or exponential backoff。セッション ID をログイン時に再生成。

## A08: Software and Data Integrity Failures（ソフトウェアとデータの整合性の失敗）

- [ ] CI/CD パイプラインが保護されているか
- [ ] 依存関係の整合性が検証されているか（lockfile）
- [ ] 署名されていないデータの逆シリアライゼーションを行っていないか

**対策パターン:** lockfile をコミットし CI で `--frozen-lockfile` を使用。CI/CD の権限を最小限に制限。

## A09: Security Logging and Monitoring Failures（セキュリティログと監視の失敗）

- [ ] 認証の成功・失敗がログに記録されているか
- [ ] 認可の失敗がログに記録されているか
- [ ] ログに機密情報が含まれていないか
- [ ] アラートが適切に設定されているか

**対策パターン:** 認証イベントをログ記録。ログから機密情報をマスキング。異常アクセスパターンのアラート設定。

## A10: Server-Side Request Forgery (SSRF)

- [ ] サーバーサイドの URL リクエストが制限されているか
- [ ] 内部ネットワークへのリクエストが防止されているか
- [ ] リダイレクト先が検証されているか

**対策パターン:** URL を許可リスト方式で検証。内部ネットワーク IP 範囲へのリクエストを拒否。
