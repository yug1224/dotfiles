# 非機能要件チェックリスト

アーキテクチャ設計・レビュー時に確認すべき品質特性の一覧。

## Availability（可用性）

- [ ] 単一障害点（SPOF）が存在しないか
- [ ] リトライ戦略が適切か（指数バックオフ、ジッター、上限回数）
- [ ] フォールバック / グレースフルデグラデーションが設計されているか
- [ ] ヘルスチェックが実装されているか
- [ ] タイムアウトが適切に設定されているか
- [ ] Circuit Breaker が必要な外部連携に導入されているか
- [ ] データバックアップとリストア手順が定義されているか

## Observability（可観測性）

- [ ] 構造化ログが出力されているか（トレース ID、ユーザー ID、操作種別）
- [ ] ビジネスメトリクスが計測されているか（成功率、処理時間、キュー深度）
- [ ] 分散トレーシングが設定されているか
- [ ] エラーアラートの閾値が定義されているか
- [ ] ダッシュボードに主要指標が可視化されているか
- [ ] ログレベルが適切に使い分けられているか（ERROR / WARN / INFO / DEBUG）

## Modifiability（変更容易性）

- [ ] モジュール間の結合度が低いか（インターフェース依存、DI 活用）
- [ ] 依存の方向が内側 → 外側か（クリーンアーキテクチャの原則）
- [ ] 変更の波及範囲が予測可能か
- [ ] 拡張ポイントが設計されているか（Strategy パターン、Plugin 機構等）
- [ ] フィーチャーフラグで段階的リリースが可能か
- [ ] 設定値がハードコードされず外部化されているか

## Performance（性能）

- [ ] N+1 クエリが存在しないか
- [ ] 適切なインデックスが設計されているか
- [ ] キャッシュ戦略が定義されているか（TTL、invalidation、cache stampede 対策）
- [ ] ページネーションが実装されているか（大量データの一括取得を避ける）
- [ ] バッチ処理とリアルタイム処理が適切に使い分けられているか
- [ ] 計算量（時間 / 空間）が許容範囲内か
- [ ] 非同期処理が必要な箇所で使われているか

## Security（セキュリティ）

- [ ] 認証・認可の境界が明確か
- [ ] 入力バリデーションが網羅されているか
- [ ] 機密データが暗号化 / マスキングされているか
- [ ] 最小権限の原則が遵守されているか
- [ ] テナント分離が適切か（マルチテナントの場合）
- [ ] シークレットがコードにハードコードされていないか
- [ ] CORS / CSP / セキュリティヘッダーが設定されているか

## Scalability（拡張性）

- [ ] 水平スケールが可能な設計か
- [ ] ステートレスなコンポーネント設計か（セッション状態の外部化）
- [ ] データ量増加に対する対策があるか（パーティショニング、アーカイブ）
- [ ] 同時接続数の見積もりと対策があるか

## Operability（運用性）

- [ ] デプロイがゼロダウンタイムで可能か
- [ ] ロールバック手順が定義されているか
- [ ] マイグレーション手順が安全か（データ損失リスクの評価）
- [ ] 運用手順書 / ランブックが必要な箇所に用意されているか
