# ブログ記事メタ情報・SEO ルール

タイトル、タグ、要約など記事のメタ情報を決定する際のガイドライン。

---

## タイトル

- **30-50 文字**を目安にする
- 具体的なキーワードを含める（ツール名、技術名、数字）
- 以下のパターンを使い分ける:
  - **成果型**: 「〇〇で△△が爆速になった」「〇〇を導入して□□を実現した」
  - **ハウツー型**: 「〇〇で△△を作ろう」「〇〇の始め方」
  - **ストーリー型**: 「〇〇だった半年間」「〇〇から△△へ」
  - **問いかけ型**: 読者の疑問をそのままタイトルにする
- 候補は 3 案以上出し、最終的に 1 つを選ぶ

## 絵文字（Zenn emoji フィールド）

- 記事の内容やトーンに合った絵文字を 1 つ選ぶ
- テーマと直結するもの、またはユーモアのあるものが好ましい
- 例: 技術改善 → ⚡️、キャリア → 🔄 / 🦋、DIY → 🪵

## トピック / タグ

- **3-5 個**を選定する
- 第 1 タグ: メインの技術やテーマ（検索流入の柱）
- 第 2-3 タグ: 関連技術・フレームワーク
- 残りのタグ: 文脈タグ（Advent Calendar 名、イベント名など）
- プラットフォームごとのタグ制約:
  - Zenn: `topics` フィールド、最大 5 個
  - Qiita: `tags` フィールド、最大 5 個

## 要約（OGP / description 用）

- **80-120 文字**で記事の価値を伝える
- 「誰向けに」「何を」「どんな結果が得られるか」を含める
- SNS でシェアされた際に読みたくなる一文にする

## type フィールド（Zenn）

- `tech`: 技術的な内容が中心の記事
- `idea`: キャリア、ポエム、エッセイ系の記事

## publication_name（Zenn）

- 所属組織の publication がある場合は指定する
- 個人記事の場合は省略する
