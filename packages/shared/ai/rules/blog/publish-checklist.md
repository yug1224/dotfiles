# ブログ記事 公開前チェックリスト

記事の公開前に以下の項目を順に確認する。各項目を PASS / FAIL で判定し、FAIL がある場合は修正案を提示する。

---

## 1. 基本構成

- [ ] 「はじめに」と「おわりに」（または「まとめ」）があるか
- [ ] H2 → H3 の階層が正しく、H1 は使っていないか
- [ ] 見出しの粒度が揃っているか（極端に長い/短いセクションがないか）

## 2. frontmatter

- [ ] `title` が設定されているか
- [ ] `emoji`（Zenn）/ アイキャッチ画像が設定されているか
- [ ] `topics` / `tags` が 3-5 個設定されているか
- [ ] `type`（Zenn: tech / idea）が適切か
- [ ] `published: true` になっているか（公開する場合）
- [ ] `published_at` が指定日時になっているか（予約投稿の場合）

## 3. 本文品質

- [ ] 誤字・脱字がないか
- [ ] 同じ語尾（「〜です。〜です。〜です。」）が 3 回以上連続していないか
- [ ] 1 段落が 200 文字を大幅に超えていないか
- [ ] 太字の使いすぎ（1 段落に 3 箇所以上）がないか

## 4. コードブロック

- [ ] すべてのコードブロックに言語指定があるか
- [ ] コード内のハードコードされた秘密情報（API キー、トークン等）がないか
- [ ] コードが実際に動作するか（手元で検証済みか）

## 5. リンク・埋め込み

- [ ] 外部リンクがすべて有効か（リンク切れチェック）
- [ ] 埋め込み記法（`https://...` の単独行）が正しく機能するか
- [ ] 内部リンク（自分の過去記事への参照）が正しいか

## 6. 画像・メディア

- [ ] 画像がすべて表示されるか
- [ ] 画像の alt テキストまたはキャプションがあるか
- [ ] 画像ファイルサイズが大きすぎないか（目安: 1 枚 500KB 以下）

## 7. 機密情報・配慮

- [ ] 社内情報、NDA に抵触する内容が含まれていないか
- [ ] 個人情報（他者の氏名、連絡先等）が許可なく記載されていないか
- [ ] スクリーンショットに機密情報が写り込んでいないか
- [ ] 炎上リスクのある表現がないか（`blog/blog-review-rule.md` の観点 6 を参照）

## 8. SEO・メタ情報

- [ ] タイトルが 30-50 文字で具体的なキーワードを含むか
- [ ] 記事冒頭 3 行で「この記事で何が分かるか」が伝わるか
