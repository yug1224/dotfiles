# ブログネタ発掘ルール

ブログ記事のテーマを提案する際の観点と優先度。

---

## ネタの発掘観点

### 1. 日々の開発で詰まったこと・解決したこと

- エラーの解決、ワークアラウンドの発見
- 公式ドキュメントだけでは分からなかったこと
- Stack Overflow / GitHub Issues で見つけた知見

### 2. 技術的な意思決定

- ツール・ライブラリの選定理由
- アーキテクチャの設計判断
- Before / After で効果が示せる改善

### 3. PR・コミット履歴からの発掘

- 直近の git log や PR 一覧から、説明する価値のある変更を探す
- 大きめのリファクタリング、パフォーマンス改善、新機能開発

### 4. キャリア・働き方の変化

- 転職、ロール変更、チーム体制の変化
- 学びや気づきの言語化

### 5. イベント・登壇・読書

- カンファレンス参加レポート
- 登壇資料のブログ化（発表内容の再構成）
- 読了した技術書のレビュー

### 6. プライベート × 技術

- 趣味やDIYでの技術活用
- 個人開発プロジェクト

## 候補の評価基準

各候補に以下を付けて提案する:

| 項目           | 説明                                    |
| -------------- | --------------------------------------- |
| テーマ         | 記事タイトル案（仮）                    |
| 切り口         | どの角度から書くか                      |
| 記事タイプ     | 技術解説/体験記/チュートリアル/比較/etc |
| ターゲット読者 | 誰に読んでほしいか                      |
| 差別化ポイント | 既存記事との違い・独自の価値            |
| 必要な検証     | 書く前に確認すべきこと                  |
| 想定ボリューム | 軽め(1000字) / 標準(2000-3000字) / 重め |

## Advent Calendar・企画系への対応

- Advent Calendar の場合はカレンダーのテーマとの整合性を確認する
- 企画参加の場合はレギュレーション（文字数、タグ指定など）を事前に把握する
