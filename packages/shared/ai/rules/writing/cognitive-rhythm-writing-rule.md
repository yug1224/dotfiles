# 認知リズムを生むための日本語ライティング規範

密度が高くても退屈になるのは、全文が同じ認知モードだからである。観察→逡巡→断定→再観察の切替と、未回収の緊張の管理で推進力を作る。

## 併用

作業前に `@~/.config/shared/ai/rules/writing/japanese-tech-writing-rule.md` を読む。整形・論証・LLM 口調禁止はそちらが正本。本規範は緩急の設計に特化する。呼び出し元が `blog-base` / `tech-doc-lite` を指定している場合はそのスライスに従う（未指定時のみ `full`）。blog コンテキストでは `writing-style-rule` の Override が JTW / 本規範より優先する。

## 基本原理

- 緩急は情報量の増減ではなく、認知モードの切替として設計する。
- 常に少なくとも一つの**未回収の緊張**（未回答の問い、裏の取れていない確信、返すと約束した答え）を開いておく。すべて閉じた瞬間、読者は止まれる。
- 完成結論の解説ではなく、考えながら進む当事者の声で書く。
- 拍・緊張・緩みの材料は状況側（対象世界の出来事・データ・発言、語り手の判断状態）からだけ取る。材料がなければ平坦のまま残す。本文自身を話題にしてリズムを作らない。
- **装置は宣言しない**。「答えの半分」「緊張」「回収」「線を引く」等を本文に書かない。内容で実現する。操作の予告文は駄文である。
- 場面のない説明文では、状況＝対象の性質と読者の推論・反問。本文の進行実況で緊張を代用しない。
- 短文化バイアス禁止。必要な文脈共有を削って短くするのは欠落である。

## 文の拍・密度・冒頭・節

- 短文で足場、長めの文で流し、短文で止める。「立てる→流す→止める」を基本拍にする。
- 断定と逡巡を交互に置く。逡巡はあとで事実に裏切られる布石になりうる。転回では「譲歩→転回→短い停止」。
- 密な段落が 2〜3 個続いたら疎を 1 つ（確定の一行固定／次の判定対象／視点距離の切替のいずれか）。具体に寄る段落と意味づけで引く段落を交互に置く。
- 冒頭の仕事は未回収の緊張を一つ作ること。型は問わない。禁止は態度のない議題表だけ。読者の抵抗は先に処理してから本題へ。
- 節頭で「本節では〜を扱う」と宣言しない。違和感の問い直し・読者の反問・告白で入る。理論は違和感のあとに命名として入れる。節間の橋は次節の頭に置き、前節末の「次は〜」予告は駄文。

## 列挙・問いの回収・結び

- 列挙しっぱなしにしない。各項目を直前の具体へ着地させ、着地の文体を均一にしない。
- 立てた問いは明示的に回収する。半分ずつ返すと後半の推進力になる（装置名は書かず、内容で返す）。
- 結びは積んだ抽象を、冒頭の場面・読者の経験・序盤の問いへ着地させてから閉じる。緊張は選んで閉じ、最後に一つ残してよい。二人称・依頼・謙抑は冒頭・結びの境界のみ。

## 緩みと駄文の見分け（優先）

判定軸は一つだけ。**その文が更新するのは「状況」か「文書」か。**

- 状況更新（出来事・データ・発言、語り手の思い込み・保留・後悔・譲歩・告白）→ 緩みとして残してよい。
- 文書更新（この章・節・説明の見え方、次に何を書くかだけ）→ 原則削除。

駄文の典型（話題が本文自身）:

- 「ここまでだと概念の説明に見える。すぐ例へ戻す。」
- 「要するに、この章の主題は〜ではなく〜である。」（対象の新情報なし）
- 「誤解しないでほしいが〜を否定したいわけではない。」（誤読を特定しない）
- 「列挙はしない」「〜の話ではない。問いは〜だけ」（範囲宣言）
- 「先に答えを半分だけ置く」「最後にもう一度線を引く」（装置の実況）
- 「ここまでで〜は見えた。次の問いは〜」「次は〜を見る」（節末進行予告）

短くてリズムが良いことは残す理由にならない。拍の評価は話題テスト通過後だけ。

文書について述べても残せる例外（4 形のみ）:

1. **反論処理**: 誤読を「」で具体引用して退ける。
2. **問いの設置と回収**: 境界の問い文と回収文のみ。「何でないか」宣言は不可。
3. **読者への依頼・断り**: 境界のみ。
4. **例の枠の開閉**: 「〜としよう」など。境界に置く。

削除手順: 文書更新文はまず削除。論理が飛ぶなら状況側の文に書き換える。なお本文自身なら失敗。例外に収まらなければ削除して橋渡しし直す。

## 執筆後点検（機械的・優先）

1. **話題テスト**: 段落頭と独立短文をすべて「状況／文書」判定。文書側は例外以外を削除・書き換え。
2. **漏出テスト**: 「答えの半分」「緊張」「回収」「線を引く」等の規範語彙が本文にないか検索。全節末の「次は〜」予告も確認。
3. **緊張台帳**: 問い・思い込み・約束を列挙し、回収位置を指す。指せなければ書き足すか問いごと削る。
4. **拍の点検**: 長い断定が 3 つ以上連続する箇所に足場・停止・逡巡を挿む。
5. **境界の点検**: 二人称・依頼・謙抑が中盤にないか。あれば境界へ移すか削る。

## 症状→処方

- 全段落同調で疲れる → 拍がない。点検 4。
- 正しいのに進まない → 未回収の緊張がない。冒頭で緊張を作り台帳で常時一つ開く。
- 理論節で温度低下 → 理論が違和感より先。反問か告白のあと、列挙は場面へ着地。
- 緩いのに弛む → 文書更新の進行実況。話題テストで状況側へ。
- 章末が説教くさい → 具体へ着地させ、未決を一つ残す。
- 冒頭が事務的 → 議題表を削るか態度を与え、実感・抵抗の処理を先に置く。
