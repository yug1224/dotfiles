# チケット情報取得ルール

引数にチケット ID または URL が渡された場合のみ、以下の手順でチケット情報を取得する。引数がなければスキップする。

## ローカルアダプタ（任意）

同ディレクトリの `ticket-retrieval-rule.local.md` を Glob で確認する。存在する場合のみ Read し、プロバイダ固有の手順はそちらを優先する。

Provider-specific adapters（外部チケット MCP 等、チケット ID プレフィックスのマッピング）は `.local` に置く（`ticket-retrieval-rule.local.md` が存在する場合）。

## チケットシステムの判別と取得方法

| パターン                | 判別条件                   | 取得方法                                                  |
| ----------------------- | -------------------------- | --------------------------------------------------------- |
| **GitHub Issues / PR**  | URL に `github.com` を含む | `gh` CLI または GitHub MCP                                |
| **その他の URL**        | 上記以外の URL             | URL フェッチで内容を取得                                  |
| **ID のみ（URL なし）** | URL 形式でない文字列       | ユーザーに補足情報を確認する、または local アダプタに従う |

## 取得する情報

- チケットのタイトル・目的
- 背景・要件・受け入れ条件（あれば）
- 関連する技術的コンテキスト
